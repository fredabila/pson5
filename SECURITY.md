# Security Policy

## Supported Scope

Security reports are especially relevant for:

- authentication and authorization logic
- provider access and prompt redaction
- tenant isolation
- profile export/import behavior
- storage backends and data leakage risks

## Reporting A Vulnerability

Do not open a public GitHub issue for a suspected vulnerability.

Instead:

- contact the maintainers privately
- include reproduction details
- include affected package or route
- include impact assessment if known

If the issue involves profile or tenant data exposure, say so explicitly.

## Response Goals

- acknowledge receipt quickly
- validate impact before public disclosure
- ship a fix or mitigation before publishing details

## Disclosure Guidance

Please avoid sharing exploit details publicly until maintainers confirm remediation or a coordinated disclosure plan.
