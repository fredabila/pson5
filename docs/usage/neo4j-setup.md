# Neo4j setup — three paths

> The PSON5 knowledge graph lives inside every `.pson` profile. Neo4j is an **optional** mirror, useful when you want to visualise the graph, run cross-profile queries, or plug into downstream graph tooling. This page covers three setup paths; pick the one that matches what you have installed.

## tl;dr — decide in 30 seconds

| You have… | Use |
| --- | --- |
| **Docker Desktop** running | [**Local**](#1-local-via-docker-one-command) — one command, zero cost |
| No Docker, want zero install | [**Aura free cloud**](#2-neo4j-aura-cloud-free-tier) — 5-minute signup |
| Want to look at the graph **right now** with no infra | [**Built-in HTML viewer**](#3-no-infra-graphhtml) — already generated by the demos |

## 1. Local, via Docker (one command)

The repo ships a ready-to-run Compose file plus a wrapper that starts the container, waits for it to be ready, and wires up the PSON5 CLI automatically.

### Prerequisites

- Docker Desktop installed and running (`docker --version` should work).

### Start it

**macOS / Linux / Git Bash / WSL:**
```bash
./scripts/neo4j-up.sh
```

**Windows PowerShell:**
```powershell
.\scripts\neo4j-up.ps1
```

The script:

1. Pulls and starts `neo4j:5.24-community` on ports 7474 (HTTP) and 7687 (Bolt).
2. Waits up to 2 minutes for the instance to become healthy.
3. Writes `<store>/config/neo4j.json` so the PSON5 SDK, API, and CLI pick it up automatically.

When it finishes:

```
Neo4j Browser    http://localhost:7474
Credentials      neo4j / pson5-local-dev
Bolt URI         bolt://localhost:7687
```

Open Neo4j Browser and log in with those credentials.

### Sync a profile

After you've run one of the demos (either `examples/josh-tech-persona/` or the richer `examples/claude-driven-persona/`) there is a `profile.json` under the example's `output/` directory. Push it into Neo4j:

```bash
node scripts/sync-profile-to-neo4j.mjs examples/claude-driven-persona/output/profile.json
```

The sync:

- Creates a `:PsonProfile` node for the profile and a `:PsonUser`-`OWNS_PROFILE`->profile edge.
- Detach-deletes any previous `:PsonNode` records tagged with this `profile_id` (safe to re-run).
- Writes every knowledge-graph node as `:PsonNode {profile_id, node_id, type, label, data_json}`.
- Writes every edge as `:PSON_EDGE {profile_id, edge_id, edge_type, data_json}`.
- Prints a starter Cypher query so you can inspect the result immediately.

Paste the printed Cypher into Neo4j Browser:

```cypher
MATCH (p:PsonProfile { profile_id: "pson_…" })-[:HAS_NODE]->(n:PsonNode)
OPTIONAL MATCH (n)-[r:PSON_EDGE]->(m:PsonNode)
RETURN p, n, r, m;
```

### Stop it

```bash
docker compose -f scripts/docker-compose.neo4j.yml down
```

Add `--volumes` to also drop the data volume.

### Using PSON5 CLI directly

Once `<store>/config/neo4j.json` is populated, every PSON5 surface can use it:

```bash
pson neo4j-status --store .pson5-store
pson neo4j-sync <profile_id> --store .pson5-store
```

## 2. Neo4j Aura (cloud, free tier)

Zero install. Aura's free tier gives you 200k nodes / 400k relationships — more than enough for any PSON5 workload you'll start with.

### Sign up

1. Go to [neo4j.com/cloud/aura-free](https://neo4j.com/cloud/aura-free).
2. Create a free account, then create a new AuraDB Free instance. Pick any region close to you.
3. **Download the generated password.** Aura shows it exactly once — keep it safe.
4. Once the instance is `Running`, grab the Bolt URI (looks like `neo4j+s://<id>.databases.neo4j.io`).

### Point PSON5 at it

```bash
# Option A — env vars (winning over stored config)
export PSON_NEO4J_URI="neo4j+s://<id>.databases.neo4j.io"
export PSON_NEO4J_USERNAME=neo4j
export PSON_NEO4J_PASSWORD="<your-aura-password>"

pson neo4j-status --store .pson5-store

# Option B — persist it so you don't need env vars every time
pson neo4j-wizard --store .pson5-store
#   → URI:      neo4j+s://<id>.databases.neo4j.io
#   → Username: neo4j
#   → Password: <your-aura-password>
#   → Database: (leave blank)
```

### Sync a profile

Identical to the local path:

```bash
node scripts/sync-profile-to-neo4j.mjs examples/claude-driven-persona/output/profile.json
```

### Explore

Aura's web UI has a built-in Neo4j Browser — the `MATCH` query above works the same way there.

## 3. No infra — `graph.html`

If you just want to **see** the graph without setting up any database, every demo already produces a self-contained interactive viewer. Open this in a browser:

```
examples/claude-driven-persona/output/graph.html
```

Features:

- D3 force-layout with drag-to-reposition and scroll-to-zoom
- Observed/inferred/simulated colour grammar matching the rest of the stack
- Sidebar with node-type distribution, the three agent contexts produced during the run, and the three scenario simulations inline
- Zero JavaScript runtime dependencies beyond D3 from CDN — works offline once loaded

Use this for sharing a screenshot, embedding in a post-mortem, or just seeing what your profile looks like before you commit to a database.

### Paste into a real Neo4j

The same demos also produce a `graph.cypher` file with ready-to-paste statements. If you have a Neo4j instance (either of the paths above, or one you already run), you can paste it directly into Neo4j Browser:

```bash
cat examples/claude-driven-persona/output/graph.cypher
```

## What ends up in Neo4j

A single `syncKnowledgeGraphToNeo4j(...)` call produces the following graph shape:

```
(:PsonUser { user_id })
    │ OWNS_PROFILE
    ▼
(:PsonProfile { profile_id, user_id, pson_version, revision, updated_at })
    │ HAS_NODE
    ▼
(:PsonNode { profile_id, node_id, type, label, data_json })
    │
    │ PSON_EDGE { profile_id, edge_id, edge_type, data_json }
    ▼
(:PsonNode { … })
```

Every node and edge is tagged with `profile_id`, so cross-profile queries are always explicit. Re-syncing the same profile detach-deletes the old profile-scoped nodes first, so the graph always reflects the current `.pson` snapshot.

## Troubleshooting

**"docker was not found on PATH"**  
Install Docker Desktop or use Aura (path 2). The script prints this message with the links.

**"docker is installed but the daemon is not reachable"**  
Start Docker Desktop, wait ~30s for it to finish booting, then retry.

**"Neo4j did not become healthy within 120s"**  
Check the logs: `docker compose -f scripts/docker-compose.neo4j.yml logs`. First-time boots on slow machines can need the full timeout; re-running `./scripts/neo4j-up.sh` will resume waiting.

**Sync fails with "Neo4j is not configured"**  
Either the stored config is missing or the env vars aren't set. Run `pson neo4j-status --store .pson5-store` and fix whichever path you're using.

**Aura connection refused / `ServiceUnavailable`**  
Confirm the instance is `Running` in the Aura console. Aura instances **pause after 3 days of inactivity** on the free tier — you may need to resume yours.
